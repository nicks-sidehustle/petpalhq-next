# Accessibility and UX Audit

## Current concerns

- Nav text is too small.
- Some green/gold text may be too low contrast.
- Guide body is too dense.
- Touch targets need review.
- Focus states need review.
- Mobile guide flow likely has too much before the article.

## Minimum standards for next pass

- Body text: 16px minimum, preferably 17–18px for article prose.
- Nav text: 15–16px minimum.
- Tap targets: about 44px high where possible.
- Contrast: avoid gold text on cream for small text.
- Links: visible underline or strong affordance in body copy.
- Focus states: visible keyboard focus.
- Mobile: primary content visible quickly.

## Guide page UX requirements

The guide page should start with:

1. Title
2. One-sentence answer
3. Best picks / decision module
4. Comparison table
5. Buying advice
6. FAQ

Avoid:

- Large irrelevant hero images.
- Multiple trust/disclosure blocks stacked together.
- Long paragraphs before product decisions.
- Tiny metadata text as primary orientation.
